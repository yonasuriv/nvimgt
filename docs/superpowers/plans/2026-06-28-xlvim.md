# nvimGT Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use `superpowers:subagent-driven-development` (recommended) or `superpowers:executing-plans` to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Produce a working nvimGT Neovim configuration that merges LazyVim’s base with AstroDark theme, an AstroNvim-style top tabline, and an NvChad-style bottom statusline.

**Architecture:** LazyVim starter layout bootstraps `lazy.nvim` and imports `lazyvim.plugins`. Local `lua/plugins/` files add `astrotheme`, override the LazyVim colorscheme, re-section `lualine.nvim` for the statusline, disable `bufferline.nvim`, and add a `heirline.nvim` tabline.

**Tech Stack:** Neovim 0.11+, `lazy.nvim`, LazyVim distribution, `AstroNvim/astrotheme`, `nvim-lualine/lualine.nvim`, `rebelot/heirline.nvim`, `echasnovski/mini.icons`.

## Global Constraints

- Keep LazyVim as the plugin distribution; do not install AstroNvim/NvChad core plugins.
- Use `astrodark` as the active colorscheme.
- Reuse existing LazyVim defaults where possible (options, keymaps, autocmds structure).
- All new plugin spec files live under `lua/plugins/`.
- Branding and comments in local files must use **nvimGT**, not LazyVim/AstroNvim/NvChad.

### Task 2: Add AstroDark colorscheme

**Files:**
- Create: `lua/plugins/colorscheme.lua`

**Interfaces:**
- `astrotheme` is loaded non-lazy with high priority.
- LazyVim colorscheme override sets `colorscheme = "astrodark"`.

- [ ] **Step 1: Write `lua/plugins/colorscheme.lua`**

```lua
return {
  -- nvimGT colors: AstroNvim AstroDark
  {
    "AstroNvim/astrotheme",
    lazy = false,
    priority = 1000,
    opts = {},
  },
  -- Tell LazyVim to use astrodark
  {
    "LazyVim/LazyVim",
    opts = {
      colorscheme = "astrodark",
    },
  },
}
```

- [ ] **Step 2: Test colorscheme load**

Run:

```bash
nvim --headless -c "colorscheme astrodark" -c "qa"
```

Expected: no error, `vim.g.colors_name == "astrodark"`.
